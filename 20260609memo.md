Airgraph Gitアップロード & APIキー管理手順書

前提条件
GitHubなどのGitホスティングサービスで、空のリポジトリ（例：Airgraph）をWeb上で作成しておいてください。この際、Initialize this repository with のオプション（READMEや.gitignore、Licenseの追加）は**すべて未チェック（空リポジトリ）**にしておくとスムーズです。
リモートリポジトリのURL（例：https://github.com/ユーザー名/Airgraph.git）を手元に控えておきます。

本ドキュメントでは、APIキーを安全に保護しながら、AirgraphプロジェクトをGitにアップロードする手順を解説します。

1. APIキーの保護設定（.envの利用）
APIキーなどの機密情報は、ソースコードに直接書かずに .env ファイルに切り出し、Gitの追記対象から除外（.gitignore）します。

ステップ 1: .env プレースホルダーの作成
プロジェクトのルートフォルダ（c:\Users\y86as\研究20260528）に .env という名前のファイルを作成し、APIキーを書き込みます。

.env の中身:

text

GEMINI_API_KEY=あなたの実際のAPIキー
# 他にキーがあればここに追加します
ステップ 2: .gitignore への登録
.gitignore ファイルを開き、以下の行を追加して保存します。これにより、.env ファイルがGitにアップロードされるのを防ぎます。

.gitignore の中身:

text

.env
# 他の不要なキャッシュや仮想環境フォルダもついでに除外します
__pycache__/
*.pyc
.venv/
venv/
11:16 ok
ステップ 3: プログラム側での読み込み（参考）
Pythonプログラム内でAPIキーを利用する際は、os.environ や python-dotenv ライブラリを使用して環境変数から取得するように書き換えます。

python

import os
# 必要に応じて: from dotenv import load_dotenv; load_dotenv()
api_key = os.environ.get("GEMINI_API_KEY")

2. Gitへのアップロード手順
APIキーの保護設定が終わったら、以下のコマンドをご自身のPowerShell等のターミナルで順に実行します。

① Git初期化と最初のコミット（Airgraph本体）
powershell

# 1. プロジェクトフォルダに移動
cd c:\Users\y86as\研究20260528
# 2. リポジトリの初期化
git init
# 3. .env が除外されているか確認 (一覧に .env が無ければOK)
git status
# 4. すべてのファイルをステージングに追加
git add .
# 5. 初回コミットを作成
git commit -m "Initial commit: Airgraph core"
# 6. メインブランチの名前を main に設定
git checkout -b main
# 7. リモートリポジトリ（GitHubなど）の登録
# (※ URLはご自身で作成したリポジトリのものに書き換えてください)
git remote add origin https://github.com/ユーザー名/Airgraph.git
# 8. リモートに送信
git push -u origin main
3. 実験結果をブランチに分けてアップロードする手順
本体を main ブランチに上げたら、実験ごとにブランチを分けてコミットします。

② 1回目の実験結果（そのままの結果）
powershell

# 1. 1回目用のブランチを作成して切り替える
git checkout -b experiment-run1
# 2. 1回目の結果ファイルをフォルダ内に配置する
# 3. 変更をステージングしてコミット
git add .
git commit -m "Add results of 1st run (baseline)"
# 4. リモートにプッシュ
git push -u origin experiment-run1
③ 2回目の実験結果（埋め込みモデルの変更）
powershell

# 1. 一度 main ブランチに戻る
git checkout main
# 2. 2回目用のブランチを作成して切り替える
git checkout -b experiment-run2-embedding-change
# 3. 埋め込みモデルを変更したコード、および2回目の結果ファイルを配置する
# 4. 変更をステージングしてコミット
git add .
git commit -m "Change embedding model and add results of 2nd run"
# 5. リモートにプッシュ
git push -u origin experiment-run2-embedding-change
