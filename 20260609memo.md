前にあるやつは、もしかしたら、再現実験でそのひとからとってきたコードでも、私のリポジトリが更新される？

とても重要な疑問ですね！結論から言うと、勝手に元の作者（他の人）のリポジトリが更新されたり、あなたのコードが元の作者のリポジトリに書き込まれたりすることはありません。

仕組みを説明します：

取得した時（cloneした時）の元の接続情報 もし元のコードを git clone で持ってきた場合、そのフォルダのGitには、元の作者のリポジトリのURL（例: https://github.com/元の作者/AriGraph.git）が「origin」という名前で登録されています。

あなたがプッシュ（push）しようとした場合 あなたがその状態で git push しようとしても、元の作者のリポジトリへの書き込み権限（アクセス権）がないため、必ずエラーになり拒否されます。

自分用のリポジトリにするには あなたが新しく作ったご自身のリポジトリ（例: https://github.com/あなたのユーザー名/AriGraph.git）にアップロードするには、Gitに登録されている「接続先（リモート）」をご自身のリポジトリURLに上書き登録する必要があります。

確認と上書き登録の方法
まずは AriGraph フォルダに移動し、現在の登録情報を確認しましょう。

powershell


cd AriGraph
git remote -v
もし、これで元の作者のURLが表示された場合は、以下のコマンドでご自身のリポジトリURLに上書きします。


git remote set-url origin https://github.com/あなたのユーザー名/AriGraph.git
